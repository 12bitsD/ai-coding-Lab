# PRD 启动 Prompt

> 快速启动一个新产品的 PRD 流程，适合 0-1 阶段 MVP。

## 使用方法

复制下面的 prompt，替换 `[产品名称]` 和 `[背景]`，粘贴给 AI（Claude/GPT 均可）。

## Prompt

```markdown
我想做一个 [产品名称]。

背景：[简单说明为什么想做]

帮我做 PRD，用苏格拉底式提问的方式，每次只问我一个问题，给出 2-3 个选项让我选，不要一次问太多。
```

## 示例

```markdown
我想做一个 AI 学习路径规划器。

背景：我发现自学新领域时，不知道该先学什么后学什么，想做个工具帮我拆解学习路径。

帮我做 PRD，用苏格拉底式提问的方式，每次只问我一个问题，给出 2-3 个选项让我选，不要一次问太多。
```

## 流程总览

```
你提需求 → AI 问澄清 → 你答 → AI 出选项 → 你选 → AI 更新 PRD → 循环直到完成
```

## 后续阶段的触发 Prompt

如果 AI 没有自动进入下一阶段，可以用这些 prompt 触发：

### Step 2：定义核心概念

```markdown
继续定义这个产品的核心概念，包括：
1. 有哪些核心实体
2. 每个实体有哪些状态
3. 状态怎么流转

每个决策点给我选项让我选。
```

### Step 3：定页面结构

```markdown
现在定义页面结构，列出这个产品需要哪些页面，每个页面的核心功能是什么，页面之间怎么跳转。
```

### Step 4：逐页细化

```markdown
现在开始逐页细化，从 [页面名] 开始。

要求：
1. 先画出页面的 ASCII 布局图
2. 然后逐个区域、逐个元素问我设计选择
3. 每次只问一个问题，给 2-3 个方案让我选
4. 要覆盖正常状态、空状态、加载状态、错误状态
```

### Step 5：定 AI/Agent 接口

```markdown
现在定义这个产品需要的 AI/Agent 接口，包括：
1. 触发时机
2. 输入结构（JSON）
3. 输出结构（JSON）
4. Agent 具体做什么

逐个 Agent 来，有选择让我做决策。
```

### Step 6：定数据结构

```markdown
现在定义数据存储结构，每个实体需要哪些字段，字段类型是什么，数据来源是用户输入还是系统生成还是 AI 生成。
```

### Step 7：输出 PRD

```markdown
更新一下最新的 PRD
```

### Step 8：检查遗漏

```markdown
回顾一下我们所有的聊天记录，看下 PRD 里有没有遗漏或者没对上的结论
```

## 快捷指令

| 场景 | 你说 |
|------|------|
| 推进流程 | `继续` |
| 不想纠结 | `你定` / `这个不重要` |
| 获取文档 | `更新 PRD` |
| 修改决定 | `XX 改成 YY` |
| 检查遗漏 | `回顾一下有没有遗漏` |
| 跳过当前页 | `跳过这页` |

## 时间参考

| 阶段 | 时长 | 你的操作 |
|------|------|----------|
| 启动+澄清 | 5 min | 回答 3-5 个问题 |
| 定义概念 | 5 min | 选 1-2 次 |
| 定页面结构 | 5 min | 确认 1 次 |
| 逐页细化 | 每页 15-30 min | 每页选 10-20 次 |
| 定 AI 接口 | 10 min | 确认 2-3 次 |
| 定数据结构 | 5 min | 确认 1 次 |
| 输出 PRD | 1 min | 说 "更新 PRD" |

**3 页产品 MVP**：约 2 小时，你做选择约 50-80 次
