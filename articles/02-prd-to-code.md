# 从PRD到代码

> 还没写。先把坑占上。

## 这篇要聊什么

PRD写完了，下一步是让AI根据PRD生成代码。

我会试三种方式：

### 方式A：一次性生成

把整个PRD丢给Claude/GPT，让它一次性生成完整项目。

预期好处：整体性好，文件之间引用一致，一次搞定省事。

预期问题：容易超出上下文限制，出问题难以定位。

### 方式B：按页面拆分

把PRD按页面拆分，逐个生成再组装。

预期好处：单次生成更可控，出问题容易定位。

预期问题：文件之间引用可能不一致，需要手动组装。

### 方式C：骨架优先

先让AI生成项目结构和骨架（类型定义、空组件、路由），再填充细节。

预期好处：架构清晰，填充时有上下文，更接近人类开发流程。

预期问题：步骤多，需要多轮对话。

## 目前的猜测

我赌C会赢。原因：骨架优先的方式让AI在填充细节时有足够的上下文——它知道项目结构长什么样，知道其他文件怎么命名，知道类型定义在哪。

但这只是猜测。得实际跑一遍才知道。
