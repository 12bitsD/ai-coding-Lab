# 从 PRD 到代码

> 待写

## 预告内容

PRD 写完了，下一步是让 AI 根据 PRD 生成代码。

我会尝试几种方式：

### 方式 A：一次性生成

把整个 PRD 丢给 Claude/GPT，让它一次性生成完整项目。

**预期优点**：
- 整体性好，文件之间引用一致
- 一次搞定，省事

**预期缺点**：
- 容易超出上下文限制
- 出问题难以定位

### 方式 B：按页面拆分

把 PRD 按页面拆分，逐个生成再组装。

**预期优点**：
- 单次生成更可控
- 出问题容易定位

**预期缺点**：
- 文件之间引用可能不一致
- 需要手动组装

### 方式 C：骨架优先

先让 AI 生成项目结构和骨架（类型定义、空组件、路由），再填充细节。

**预期优点**：
- 架构清晰
- 填充时有上下文
- 更接近人类开发流程

**预期缺点**：
- 步骤多
- 需要多轮对话
