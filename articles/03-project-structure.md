# AI-friendly 的项目结构设计

> 待写

## 预告内容

如果 AI 要成为主力开发者，项目结构就得为 AI 优化。

本篇将探讨：

### 目录组织

- 按功能还是按类型组织？
- AI 更容易理解哪种结构？
- 如何减少 AI 的上下文负担？

### 命名规范

- 文件名、变量名、函数名的命名模式
- AI 能从命名推断出什么信息？
- 什么样的命名让 AI 更容易定位代码？

### 模板和约定

- 新建页面/组件的标准模板
- 状态管理的统一模式
- API 调用的统一封装

### AI 维护指引

- 如何在代码库中嵌入给 AI 看的指引？
- 注释 vs 专门的文档 vs README
- 让 AI "知道" 项目的规范
